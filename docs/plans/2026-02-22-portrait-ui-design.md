# 无极 - 竖屏 UI 设计文档

**日期**: 2026-02-22
**状态**: 已批准，待实施

---

## 背景

游戏当前为横屏 1280×720（16:9）。本文档描述改为**竖屏 720×1560（9:19.5）**的 UI 重设计方案，以适配现代主流手机（iPhone / 主流 Android）竖握操作。

---

## 核心布局原则

### 屏幕分区

```
┌──────────────┐  ← 0
│              │
│  游戏视口    │  上 60% = 720 × 936px
│              │
├──────────────┤  ← 936px
│              │
│  操控面板    │  下 40% = 720 × 624px
│              │
└──────────────┘  ← 1560px
```

- **游戏视口区**（0% ~ 60%）：渲染游戏内容（战斗场景、地图、NPC 等）
- **操控面板区**（60% ~ 100%）：玩家操控输入（技能、摇杆、状态信息）

### 基础尺寸

```
GAME_WIDTH  = 720
GAME_HEIGHT = 1560
宽高比       = 9:19.5
Scale Mode  = Phaser.Scale.FIT（保持等比，自动居中）
```

---

## 布局常量（uiConfig.ts 新增）

```typescript
// 分区边界
VIEWPORT_RATIO  = 0.60   // 游戏视口占屏幕高度
PANEL_RATIO     = 0.40   // 操控面板占屏幕高度
PANEL_Y         = height * 0.60   // 操控面板起始 Y

// 操控面板内部
PANEL_PADDING_X = width * 0.06
PANEL_PADDING_Y = height * 0.02
```

## 各场景布局设计

### 1. gameConfig.ts

```typescript
// 修改前
const GAME_WIDTH = 1280;
const GAME_HEIGHT = 720;

// 修改后
const GAME_WIDTH = 720;
const GAME_HEIGHT = 1560;
```

---

### 2. BattleScene（战斗场景）

```
┌──────────────────────┐  0%
│ [回合数] [标题] [HP] │  顶栏 0~8%
├──────────────────────┤  8%
│                      │
│   妖异区（上方）     │  战斗区 8%~60%
│   ──── 战场中线 ──── │
│   残魂（下方）       │
│   血条 / 状态图标    │
├──────────────────────┤  60% ← 分区线
│  战斗日志 / 技能触发 │  信息区 60%~72%
├──────────────────────┤  72%
│                      │
│  [技能1] [普攻] [技2]│  操作区 72%~100%
│   CD环    持续攻击    CD环  │
└──────────────────────┘  100%
```

**关键变更：**
- 战斗场景压缩到上 60%，战场中线从 `height*0.55` → `height*0.30`
- 妖异列布局从横排改为 2 列竖排（适配更窄的视口）
- 操控区技能按钮移至下方面板，不再覆盖战场
- HP 顶栏保持不变，仅纵向比例调整

---

### 3. MapScene（气脉地图）

```
┌──────────────────────┐  0%
│  气脉区域名 / 进度   │  顶栏 0~8%
├──────────────────────┤  8%
│                      │
│  [气穴] [气穴]       │  节点区 8%~60%
│  [气穴] [气穴]       │  2 列 Grid 布局
│  [气穴]              │  可滚动
│                      │
├──────────────────────┤  60%
│  玩家状态            │  状态区 60%~78%
│  HP ████  五行属性   │
├──────────────────────┤  78%
│  [背包]    [前进]    │  操作区 78%~100%
└──────────────────────┘  100%
```

**关键变更：**
- 节点从横排 3 列改为 **2 列 Grid**，纵向展开
- 玩家状态栏移至下方面板
- 背包 / 前进按钮固定在底部

---

### 4. RewardScene（奖励选择）

```
┌──────────────────────┐  0%
│  选择器物            │  标题区 0~8%
│  "五行之力已凝练..."  │
├──────────────────────┤  8%
│  ┌──────────────┐   │
│  │  器物卡片 1  │   │  卡片区 8%~88%
│  └──────────────┘   │  单列纵向排列
│  ┌──────────────┐   │  可滚动
│  │  器物卡片 2  │   │
│  └──────────────┘   │
│  ┌──────────────┐   │
│  │  器物卡片 3  │   │
│  └──────────────┘   │
├──────────────────────┤  88%
│       [跳过]         │  操作区 88%~100%
└──────────────────────┘  100%
```

**关键变更：**
- 所有硬编码像素值（`y: 60`、`cardWidth: 200` 等）全部替换为 `%` 计算
- 卡片从横排 3 张改为**纵向单列**
- 卡片宽度 = `width * 0.85`，高度 = `height * 0.22`

---

### 5. MenuScene（主菜单）

基本已是 `%` 布局，微调垂直间距即可：

- 标题区压缩，留出更多空间给按钮
- 五行元素圆圈改为横排紧凑布局
- 按钮宽度改为 `width * 0.7`（竖屏更宽）

---

## 改动优先级与工作量

| 文件 | 改动量 | 优先级 | 说明 |
|------|--------|--------|------|
| `gameConfig.ts` | 极小 | P0 | 改两个数字，其余自动适配 |
| `uiConfig.ts` | 小 | P0 | 增加分区常量 `PANEL_Y` 等 |
| `RewardScene.ts` | 大 | P0 | 修复全部硬编码值 + 卡片纵向排列 |
| `BattleScene.ts` | 中 | P1 | 战斗区压缩到上60%，按钮移至下方 |
| `MapScene.ts` | 中 | P1 | 节点改为2列竖排，状态栏移底部 |
| `MenuScene.ts` | 小 | P2 | 微调间距和按钮宽度 |

## 与沙盒改造的关系

本竖屏设计与 `2026-02-22-sandbox-gameplay-design.md` 中的操控面板方向一致：

- 下 40% 操控区预留了虚拟摇杆 + 技能按钮的空间
- `PANEL_Y = height * 0.60` 作为统一的分区常量，两个改造共享
